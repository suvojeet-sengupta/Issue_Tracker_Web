## üöÄ Overview

**Issue Tracker Web** is a lightweight workspace utility that allows teams to log, track, and analyze incidents in real-time. It features a sophisticated dashboard with live metrics, a synchronized system clock, and a persistent history log‚Äîall within a beautiful, responsive interface.

### ‚ú® Key Features

| Feature | Description |
| :--- | :--- |
| **Live Dashboard** | Real-time monitoring of active nodes and incident metrics. |
| **Incident Logging** | Streamlined form for quick and accurate issue reporting. |
| **Persistent History** | Local storage-based history tracking for data persistence across sessions. |
| **Server Sync** | Integrated with WorldTimeAPI for accurate server-time synchronization. |
| **Responsive UI** | Fully optimized for mobile, tablet, and desktop viewing. |
| **Glassmorphism** | Modern UI design using Tailwind CSS for a premium look and feel. |

## üõ†Ô∏è Tech Stack

The project leverages modern web technologies to ensure speed and reliability:

## ‚öôÔ∏è Getting Started

To get a local copy up and running, follow these simple steps:

1. **Clone the repository**
   ```bash
   git clone https://github.com/suvojeet-sengupta/Issue_Tracker_Web.git
   ```

2. **Navigate to the project directory**
   ```bash
   cd Issue_Tracker_Web
   ```

3. **Open the application**
   Simply open `index.html` in your preferred web browser. No build process or server setup is required!

---

## üìñ Usage

1. **Setup Profile:** On first launch, you will be prompted to enter your advisor details (Name, CRM, Org, TL).
2. **Log Issues:** Navigate to the "New Issue" tab to report a new incident.
3. **Monitor:** Use the Dashboard to see real-time stats and the latest activity.
4. **Review:** Check the "History" tab for a full log of all recorded issues.
